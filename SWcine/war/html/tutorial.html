1. Introducción

Este tutorial pretende ser una guía sencilla para mostrar las nuevas características de HTML5.

HTML5 es la quinta versión del lenguaje HTML. Según la W3C está en modo experimental, aunque cada vez es más usado por desarrolladores Web.

La versión anterior carece de características necesarias para la creación de aplicaciones modernas. Con HTML5 ya no es necesario el uso de Flash para el desarrollo de aplicaciones que superan las habilidades de un navegador (audio, vídeo, animaciones, etc.).

2. Compatibilidad en navegadores con HTML5

HTML5 oficialmente todavía no es un estándar, por lo tanto no todos los nevegadores lo soportan completamente. Aún así la mayoría de ellos continúan añadiendo nuevas características en sus nuevas versiones.

La inmensa mayoría de las características de HTML5 ya son soportadas por todos los navegadores en sus últimas versiones.

Algo que suele preocupar es si se puede portar el código de HTML a HTML5. La respuesta es sí, HTML5 es completamente retrocompatible con HTML. Lo único que tienes que hacer para empezar a usar HTML5 es cambiar el DOCTYPE por el siguiente: 

				
	<!DOCTYPE html>

3. Nuevos elementos

<article> : define un artículo.

<aside> : define contenido aparte del contenido de la página.

<bdi> : aísla una parte del texto al que se le puede dar formato de una forma distinta al formato que hay fuera.

<comando> : define un botón de comando que el usuario puede invocar.

<details> : define detalles adicionales que el usuario puede ver o esconder.

<dialog> : define un diálogo o una ventana.

<summary> : define un encabezado visible para un elemento <details>

<figure> : específica contenido como ilustraciones, diagramas, fotos, etc.

<figcaption> : define un título para un elemento <figure>.

<footer> : define un pie de página para una sección o documento.

<header> : define un encabezado para una sección o documento.

<hgroup> : agrupa un conjunto de elementos <h1> a <h6> cuando el encabezado tiene muchos niveles.

<mark> : define un texto remarcado o resaltado.

<meter> : define una medida escalar dentro de un rango conocido.

<nav> : define links de navegación.

<progress> : representa el progreso de una tarea.

<ruby> : define una anotación ruby (para tipografía del Este de Asia).

<rt> : define una explicación/pronunciación de caracteres (para tipografía del Este de Asia).

<rp> : define qué mostrar en buscadores que no soportan anotaciones ruby.

<section> : define una sección de un documento.

<time> : define una fecha/hora.

<wbr> : define un posible salto de línea.

<audio> : define un contenido de audio.

<video> : define un vídeo o película.

<source> : define múltiples elementos de medios de <video> y <audio>.

<embed> : define un contenedor para una aplicación externa o contenido interactivo (plug-in).

<track> : define pistas de texto para <video> y <audio>.

<canvas> : se usa para pintar gráficos, sobre la marcha, utilizando scripts (normalmente JavaScripts).

<datalist> : especifica una lista de opciones predefinidas para controles de entrada.

<keygen> : define un campo generador de claves (para formularios).

<output> : define el resultado de un cálculo.

3.1. Canvas

Canvas (<canvas>) es un elemento de HTML5 que sirve para dibujar gráficos sobre la marcha. Normalmente se utiliza JavaScript para ello. El elemento <canvas> es sólo un contenedor, para dibujar es necesario utilizar scripts.

Un lienzo canvas se especifica con la etiqueta <canvas>. Es necesario añadir un id para luego poder ser referenciado por un script.

Ejemplo: 

				
	<canvas id="Canvas" width="400" height="200"
	style="border:1px solid blue;">
	</canvas>

Existen numerosas funciones para dibujar en el lienzo. Desde funciones para dibujar líneas rectas, rectángulos, círculos hasta escribir texto, pintar imágenes y hacer degradados.

volver al índice

3.2. Inline SVG

SVG (Scalable Vector Graphics) se utiliza para definir gráficos basados en vectores en formato XML. Todos los atributos y elementos en archivos SVG pueden ser animados y no pierden calidad al ser redimensionados.

La principal diferencia entre Canvas y SVG es que el primero es para dibujar gráficos 2D sobre la marcha (con JavaScript) y el segundo describe gráficos 2D en XML.

El principal inconveniente de SVG es que para renderizar gráficos complejos puede ser bastante lento. No es recomendable su uso para aplicaciones de juegos.

Ejemplo: 

				
	<svg xmlns="http://www.w3.org/2000/svg" version="1.1" height="190">
	  <polygon points="100,10 40,180 190,60 10,60 160,180"
	  style="fill:lime;stroke:purple;stroke-width:5;fill-rule:evenodd;">
	</svg>

3.3. Input

La etiqueta <input *> ya se utilizaba para insertar cajas de texto y botones. Ahora además nos permite insertar cajas de tipo “email”, calendarios, sliders, etc. Estas nuevas características permiten un control de entrada y validación mucho mejor.

Estas son los nuevos tipos de input: color, date, datetime, datetime-local, email, month, number, range, search, such, time, url y week.

3.4. Vídeo

Hasta la llegada de HTML5 la mayoría de los vídeos tenían que ser mostrados con plug-ins como Flash. HTML5 proporciona un estándar para reproducir vídeos en una página Web.

Ejemplo: 

				
	<video width="400" height="200" controls>
	  <source src="file.mp4" type="video/mp4">
	  <source src="file.ogg" type="video/ogg">
	El navegador no soporta la etiqueta video.
	</video>

El atributo controls añade los controles para el vídeo (reproducción, pausa, etc). La etiqueta source vincula a los archivos de vídeo, si hay varias el navegador utilizará la primera que tenga un formato que sea capaz de reconocer.

También existen métodos, propiedades y eventos DOM para el vídeo y audio. Estos permiten manipular elementos de vídeo y audio utilizando JavaScript.

3.5. Audio

Al igual que pasaba con los vídeos hasta ahora para reproducir archivos de audio en la Web eran necesarios plug-ins (como Flash). Pero al igual que con los vídeos HTML5 proporciona un estándar para reproducir audio sin necesidad de estos plug-ins.

Ejemplo: 

				
	<audio controls>
	  <source src="file.mp3" type="audio/mpeg">
	  <source src="file.ogg" type="audio/ogg">
	El navegador no soporta la etiqueta audio.
	</audio>

Al igual que en los vídeos el atributo controls añade los controles para, en este caso, el audio. También al igual que con los vídeos la etiqueta source vincula a los archivos de audio y si hay varias el navegador utilizará la primera que tenga un formato que sea capaz de reconocer.

5.1. Web Storage

Web Storage es una alternativa a las cookies. Las cookies guardan información en el lado del cliente, pero de forma mucho más limitada a como lo hace Web Storage.

Web Storage es más seguro y más rápido. Y los datos no se suben con cada petición al servidor, no como con las cookies que por cada petición al servidor se tenían que subir. También es posible almacenar mayor cantidad de datos.

Ejemplos:

Antes de almacenar algo hay que comprobar que el navegador soporte Web Storage. 

				
	if(typeof(Storage)!=="undefined"){
	  //localStorage y sessionStorage estan soportados
	} else {
	  //No hay soporte web storage
	}

5.2. Web Workers

Los Web Workers hacen que la multitarea sea posible con JavaScript. Antes sólo podía haber un script ejecutándose al mismo tiempo, con los Web Workers puede haber más de uno ejecutándose al mismo tiempo.

El siguiente código muestra dos botones uno para arrancar un worker y otro para pararlo. La función startWorker lo único que hace es crear un nuevo worker (si este no ha sido creado ya) y cada vez que el worker envía un mensaje lo recogemos con el evento onmessage y mostramos en resultado en result. Cuando se pulsa el botón de terminar el worker se elimina. 

				
	<p>Contador: <output id="result"></output></p>
	<button onclick="startWorker()">Empezar</button> 
	<button onclick="stopWorker()">Terminar</button>
	<br><br>
	
	<script>
	var w;
	
	function startWorker(){
		if(typeof(Worker)!=="undefined"){
	  		if(typeof(w)=="undefined"){
	    		w = new Worker("test.js");
	    	}
	  		w.onmessage = function (event) {
	    		document.getElementById("result").innerHTML=event.data;
	  		};
		}else{
			document.getElementById("result").innerHTML=
			"El navegador no soporta web workers";
		}
	}
	
	function stopWorker(){ 
		w.terminate();
	}
	</script>

El siguiente código es el contenido del fichero test.js que no es ni más ni menos que el código que ejecuta el worker en segundo plano. En este caso es simplemente un contador que después de un timeout cuenta y envía el resultado de dicha cuenta. 

				
	var i=0;
	
	function timedCount(){
		i=i+1;
		postMessage(i);
		setTimeout("timedCount()",500);
	}
	timedCount();

5.3. Web Sockets

Los Web Sockets permiten aplicaciones multiusuario en tiempo real. Proporcionan un canal de comunicación bidireccional full-duplex sobre un único socket TCP.

Para establecer una conexión el cliente tiene que mandar una petición similar a la siguiente: 

				
	GET ws://echo.websocket.org/?encoding=text HTTP/1.1 
	Origin: http://websocket.org 
	Cookie: __utma=99as 
	Connection: Upgrade 
	Host: echo.websocket.org 
	Sec-WebSocket-Key: uRovscZjNol/umbTt5uKmw== 
	Upgrade: websocket 
	Sec-WebSocket-Version: 13

El servidor contestará con una respuesta de negociación similar a la siguiente: 

				
	HTTP/1.1 101 WebSocket Protocol Handshake 
	Date: Fri, 10 Feb 2012 17:38:18 GMT 
	Connection: Upgrade 
	Server: Kaazing Gateway 
	Upgrade: WebSocket 
	Access-Control-Allow-Origin: http://websocket.org 
	Access-Control-Allow-Credentials: true 
	Sec-WebSocket-Accept: rLHCkw/SKsO9GAH/ZSFhBATDKrU= 
	Access-Control-Allow-Headers: content-type

Una vez establecida la conexión ya se puede empezar a enviar datos.

Ejemplo: 

				
	var myWebSocket = new WebSocket("ws://www.websockets.org");
	
	myWebSocket.onopen = function(evt) { alert("Connection open ..."); }; 
	myWebSocket.onmessage = function(evt) { alert( "Received Message: " + evt.data); }; 
	myWebSocket.onclose = function(evt) { alert("Connection closed."); };
	
	myWebSocket.send("Hello WebSockets!"); myWebSocket.close();

En este ejemplo creamos un websocket, la URL representa la dirección a la que queremos conectarnos. El prefijo ws:// puede ser sustituido por wss:// para establecer una conexión segura.

A los eventos onopen, onmessage y onclose se les puede asociar una función para controlar dichos eventos durante el tiempo de conexión.

Para enviar un mensaje se utiliza la función send. Para cerrar la conexión hay que utilizar la función close.

5.4. Drag and Drop

En HTML5 drag and drop (arrastrar y soltar) es parte de la norma, todos los elementos pueden arrastrarse.

Ejemplo: 

				
	<img id="elementDrag" draggable="true" ondragstart="drag(evento)" >

Para hacer un elemento arrastrable hay que poner el atributo draggable a true. 

				
	function drag(ev){
		ev.dataTransfer.setData("Text",ev.target.id);
	}

El atributo ondragstart llama a la función drag(evento). Esta función indica que datos han de ser arrastrados. El método setData establece que tipo de datos han de ser arrastrados y su valor. 

				
	function drop(ev){
		ev.preventDefault();
		var data=ev.dataTransfer.getData("Text");
		ev.target.appendChild(document.getElementById(data));
	}

La función precentDefault() evita que el navegador manipule los datos. La función getData(“Text”) permite obtener los datos. La función appendChild añade el elemento. 

				
	<div id="elementDrop" ondrop="drop(evento)"
	ondragover="allowDrop(evento)"></div>

Para indicar donde se puede soltar el elemento hay que añadir al atributo ondragover al elemento donde se quiera soltar. También hay que indicar con el atributo ondrop la función que queremos que se invoque cuando se suelten los datos.

5.5. Geolocalización

Sirve para localizar la posición del usuario.

Ejemplo: 

				
	<script>
	var x=document.getElementById("demo");
	function getLocation(){
	  if (navigator.geolocation){
	    navigator.geolocation.getCurrentPosition(showPosition);
	  }else{x.innerHTML="El navegador no soporta la geolocalicacion";}
	}
	function showPosition(position){
	  x.innerHTML="Latitude: " + position.coords.latitude + 
	  "<br>Longitude: " + position.coords.longitude; 
	}
	</script>

Este ejemplo muestra la latitud y longitud de la posición donde se encuentra el usuario. El método getCurrentPosition() obtiene la posición del usuario.
